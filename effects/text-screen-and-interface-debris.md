# Text, Screen, and Interface Debris

How text becomes visual corruption in glitchcore.

## Purpose

This file defines how text, UI elements, and interface debris function as visual corruption in glitchcore — not as readable content, but as aesthetic debris that carries emotional weight.

This is the part that helps the app stop going:
"add some text"
and start going:
"terminal residue mixed with subtitle ghosting, partial legibility, high body overlap."

The text system here is designed to make language function as visual corruption — fragmented, mediated, emotionally loaded.

---

## 1. Text as Debris Philosophy

### 1.1 Core Principle

Text in glitchcore is not for reading. It's for feeling.

The words don't need to make sense. They need to create atmosphere.

**Wrong:**
- Clean typography
- Perfect legibility
- Designed placement
- Poster layout

**Right:**
- Fragmented phrases
- Partial legibility
- Overlapping body
- Debris-like scattering

---

### 1.2 Text Functions

| Function | Description |
|----------|-------------|
| Emotional residue | Words linger like feelings |
| Mediated confession | Text as screen between people |
| Interface memory | UI elements as ghost traces |
| Terminal intimacy | Command-line vulnerability |
| Subtitle ghosting | Half-heard conversations |

## 2. Text Source Types

### 2.1 Terminal Residue

**Description:**
Command-line text, error messages, system output — the language of computers talking to themselves.

**Visual Character:**
- Monospace fonts
- Green or amber on black (classic)
- Cursor blinks
- Scroll history visible
- Errors mixed with commands

**Example Phrases:**
```
> connection lost
> retrying...
> failed
> abort? (y/n)
> user not found
> message undeliverable
```

**Best For:**
- Terminal confession
- System failure moments
- Digital loneliness
- Interface heartbreak

---

### 2.2 Chat Window Fragments

**Description:**
Pieces of conversations, DMs, chat logs — the language of mediated intimacy.

**Visual Character:**
- Bubble fragments
- Username ghosts
- Timestamps floating
- "typing..." indicators
- Read receipts

**Example Phrases:**
```
are you there?
seen 2:34am
i miss you
typing...
message not sent
user is offline
```

**Best For:**
- Soft internet heartbreak
- Digital romance
- Lonely confession
- Mediated tenderness

---

### 2.3 Subtitle Ghosting

**Description:**
Movie/TV subtitles that linger, repeat, or appear where they shouldn't — the language of media bleeding into reality.

**Visual Character:**
- Yellow or white text
- Black background bars
- Multiple layers of subtitles
- Wrong language, wrong timing
- Subtitles on faces

**Example Phrases:**
```
[ambient music]
[inaudible]
i love you
[dramatic pause]
subtitles by...
[static]
```

**Best For:**
- Media-saturated moments
- Cinematic intimacy
- Screen-life bleeding
- Narrative confusion

## 3. Text Behavior Parameters

### 3.1 Density

**Sparse (0.2-0.4):**
- Few text fragments
- Large empty spaces
- Each word carries weight
- Intimate, careful

**Moderate (0.4-0.6):**
- Regular text presence
- Balanced with image
- Natural conversation density
- Comfortable reading pace

**Dense (0.6-0.8):**
- Many text fragments
- Competing for attention
- Overwhelming, anxious
- Information overload

**Maximum (0.8-1.0):**
- Text everywhere
- Image barely visible
- Chaos, confusion
- Beautiful disaster

---

### 3.2 Legibility

**High (0.7-1.0):**
- Text is readable
- Clear typography
- Designed feel
- **Use sparingly** — risks poster design drift

**Medium (0.4-0.7):**
- Text is partially readable
- Some words clear, others blurred
- Natural degradation
- **Sweet spot** for most applications

**Low (0.2-0.4):**
- Text is barely readable
- Shapes more than words
- Atmosphere over content
- **Best for debris feel**

**None (0.0-0.2):**
- Text is pure texture
- No readable words
- Abstract text shapes
- **For pure visual effect**

---

### 3.3 Overlap Bias

**Low (0.0-0.3):**
- Text stays off body/face
- Decorative placement
- Clean separation
- **Risks poster design drift**

**Medium (0.3-0.6):**
- Some text overlaps
- Some stays clear
- Balanced composition
- **Safe default**

**High (0.6-0.9):**
- Text overlaps body heavily
- Words on faces
- Mediated intimacy
- **Best for emotional impact**

**Maximum (0.9-1.0):**
- Text covers everything
- Body as text carrier
- Total mediation
- **Extreme cases only**

---

### 3.4 Repetition

**None (0.0-0.2):**
- Each phrase unique
- No repeats
- Linear narrative
- Clean, designed

**Low (0.2-0.4):**
- Occasional repeats
- Some phrases echo
- Natural conversation
- Comfortable

**Medium (0.4-0.7):**
- Regular repetition
- Key phrases repeat
- Obsessive feeling
- Emotional emphasis

**High (0.7-1.0):**
- Constant repetition
- Same phrases everywhere
- Mantra-like
- **Best for emotional intensity**

## 4. Text Driver Behaviors

### 4.1 Text Interface Debris

**Primary Function:**
Scatters text fragments across the image like digital debris.

**Behavior:**
- Text appears randomly or semi-randomly
- Fragments float independently
- No clear reading order
- Words as visual elements

**Best Combined With:**
- Compression haze (degrades text)
- Bloom (softens text edges)
- Overprint (layers text with image)

---

### 4.2 Subtitle Ghosting

**Primary Function:**
Creates layers of subtitle text that linger and overlap.

**Behavior:**
- Subtitles appear in bands
- Multiple layers visible
- Previous subtitles fade slowly
- Text on text on text

**Best Combined With:**
- Temporal echo (subtitles echo too)
- Compression (degrades subtitle clarity)
- Scanlines (subtitle band aesthetic)

---

### 4.3 Terminal Residue

**Primary Function:**
Creates command-line text that scrolls, errors, and accumulates.

**Behavior:**
- Monospace text blocks
- Cursor blinks
- Error messages mixed with commands
- History visible

**Best Combined With:**
- Phosphor glow (CRT aesthetic)
- Scanlines (monitor feel)
- Capture degradation (old terminal)

## 6. Text Drift Detection

### 6.1 Poster Design Drift

**Symptoms:**
- Text is too legible
- Placement is too designed
- Font is too nice
- No body overlap

**Fix:**
- Reduce legibility
- Increase overlap bias
- Use worse fonts
- Add repetition

---

### 6.2 UI Mockup Drift

**Symptoms:**
- Text looks like a design mockup
- Too clean, too intentional
- Looks like Figma or Sketch
- No degradation

**Fix:**
- Add compression damage
- Reduce legibility
- Add interface corruption
- Make it feel found, not designed

---

### 6.3 Quote Graphic Drift

**Symptoms:**
- One perfect quote
- Beautiful typography
- Inspirational poster feel
- No fragmentation

**Fix:**
- Break quote into fragments
- Add multiple overlapping phrases
- Reduce legibility
- Add debris around text

---

## 7. Short Anchor Rule

**Text should feel like it leaked, not like it was placed.**
