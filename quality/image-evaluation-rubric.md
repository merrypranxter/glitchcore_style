# Image Evaluation Rubric

How to judge whether a glitchcore image actually works, where it fails, and how to fix it.

## Purpose

This file defines a structured rubric for evaluating glitchcore images.

Its job is to answer:
- Does this image actually read as glitchcore?
- What subtype is it closest to?
- What is the lead artifact driver?
- What is working?
- What is drifting?
- What needs more pressure?
- What needs less bullshit?
- Is the body still recoverable?
- Does the color have hierarchy?
- Does the corruption mean anything?

A strong evaluation system matters because glitchcore can fail in two opposite ways:
- Too little happens
- Way too much happens stupidly

This rubric is designed to catch both.

---

## 1. Core Evaluation Principle

A successful glitchcore image should feel like:
- A human figure or identity image
- Transformed by a specific failure logic
- Emotionally altered by that transformation
- Held together by palette hierarchy
- Visibly mediated by screen/media behavior
- Unstable, but not meaningless
- Damaged, but not dead
- Attractive, strange, or affecting in a way that survives the corruption

If the image is just:
- Pretty
- Glitch-adjacent
- Trendy
- Effect-stacked
- Low quality
- Random
- Generic cyberpunk
- Beauty ad with RGB seasoning

Then it scores lower.

---

## 2. Main Rubric Categories

Each image should be evaluated across these categories:

1. Figure / identity anchor
2. Artifact specificity
3. Artifact hierarchy
4. Palette architecture
5. Signal density control
6. Emotional consequence
7. Media/material specificity
8. Retrieval path clarity
9. Anti-drift integrity
10. Overall subtype coherence

Each category can be scored:
- 0 = absent / failed
- 1 = weak
- 2 = partial
- 3 = good
- 4 = strong
- 5 = excellent

**Total possible score: 50**

---

## 3. Category-by-Category Rubric

### 1. Figure / Identity Anchor

**Question:** Is there a meaningful human subject, body, face, silhouette, or identity image present?

| Score | Description |
|-------|-------------|
| 0 | No readable body or identity anchor at all |
| 1 | Barely readable subject; mostly abstract texture |
| 2 | Subject exists, but weakly; hard to recover or emotionally connect with |
| 3 | Readable body/face anchor with some loss |
| 4 | Strong subject presence, even under heavy corruption |
| 5 | Subject remains powerfully present and transformed, not erased |

**What to look for:**
- Readable face
- Silhouette
- Head angle
- Gesture
- Profile
- Body position
- Selfhood or presence

**Red flags:**
- Total abstraction
- No human stakes
- Image becomes pure texture study

---

### 2. Artifact Specificity

**Question:** Is there a clearly identifiable artifact behavior or failure mode?

| Score | Description |
|-------|-------------|
| 0 | No real artifact logic; just vague digital styling |
| 1 | Generic effect language only |
| 2 | One glitch cue is present, but weak or superficial |
| 3 | A real driver is visible, but not fully convincing |
| 4 | Specific artifact behavior clearly shapes the image |
| 5 | Artifact logic is unmistakable, integrated, and central |

**What to look for:**
- Channel split
- Temporal echo
- Compression chew
- Text debris
- Raster tear
- Contour banding
- Pixel matrix
- Capture degradation

**Red flags:**
- "Glitchy" but unspecific
- Fake preset look
- Generic RGB split with no structural effect

---

### 3. Artifact Hierarchy

**Question:** Is there a clear lead driver and sensible relationship between major and minor effects?

| Score | Description |
|-------|-------------|
| 0 | Total random effect soup |
| 1 | Many effects, no obvious lead |
| 2 | Possible lead driver, but muddied by equal-volume clutter |
| 3 | Lead driver identifiable; some support hierarchy present |
| 4 | Strong driver hierarchy with useful support layers |
| 5 | Excellent hierarchy; every support effect strengthens the lead |

**What to look for:**
- One dominant system
- 1–2 strong supports
- Minor textures staying minor
- No equal-opacity chaos

**Red flags:**
- Everything screaming equally
- Redundant artifact layers
- Visual clutter with no rank

---

### 4. Palette Architecture

**Question:** Does the color have structure, hierarchy, collision logic, and emotional function?

| Score | Description |
|-------|-------------|
| 0 | Color is dead, random, muddy, or irrelevant |
| 1 | Weak palette with no clear logic |
| 2 | Some attractive color, but little hierarchy |
| 3 | Clear palette family with decent structure |
| 4 | Strong palette hierarchy and emotionally useful collisions |
| 5 | Palette is beautiful, specific, and inseparable from the artifact behavior |

**What to look for:**
- Dominant hue family
- Collision hue
- Accent puncture
- Saturation hierarchy
- Value structure
- Palette family identity

**Red flags:**
- Clown soup
- Generic magenta/cyan with no logic
- Black-background prison
- Washed-out pastel fog
- Muddy compression death

---

### 5. Signal Density Control

**Question:** Is the image's density level intentional and well managed?

| Score | Description |
|-------|-------------|
| 0 | Density totally broken; either dead-empty or unreadable sludge |
| 1 | Density weakly controlled |
| 2 | Some level exists, but rhythm and load are messy |
| 3 | Density mostly works |
| 4 | Density feels intentional and appropriate to subtype |
| 5 | Density is exquisitely controlled — rich, readable, and pressurized in the right ways |

**What to look for:**
- Sparse / moderate / dense / overloaded / delirious
- Quiet zones
- Pressure zones
- Texture load
- Readable pile-up
- No visual lint overload

**Red flags:**
- Empty style cosplay
- No breathing room anywhere
- Mush
- All noise, no rhythm

---

### 6. Emotional Consequence

**Question:** Does the corruption create a feeling, not just a look?

| Score | Description |
|-------|-------------|
| 0 | Emotionally blank |
| 1 | Mood-adjacent but superficial |
| 2 | Some feeling implied, but weakly |
| 3 | Clear mood present |
| 4 | Strong emotional tone inseparable from the treatment |
| 5 | The corruption itself becomes emotional language |

**What to look for:**
- Dreamy
- Bratty
- Lonely
- Seductive
- Confessional
- Panicked
- Haunted
- Ecstatic
- Glamorous instability
- Internet-broken tenderness

**Red flags:**
- Cool but dead
- Trend image with no actual feeling
- Generic moody digital vibe

---

### 7. Media / Material Specificity

**Question:** Does the image feel like it comes from a specific kind of screen, file, or failure process?

| Score | Description |
|-------|-------------|
| 0 | No screen/media specificity at all |
| 1 | Vague digital feel only |
| 2 | One media cue present but weak |
| 3 | Specific medium implied |
| 4 | Material/media feel strongly shapes the image |
| 5 | The media origin feels vivid, specific, and conceptually important |

**What to look for:**
- CRT phosphor
- Subtitle residue
- Codec mush
- Screenshot fatigue
- CCTV
- Terminal glow
- Pixel lattice
- Dead notification debris
- Reposted file softness

**Red flags:**
- Empty tech styling
- Cyber aesthetic with no actual media logic
- "Retro" with no process feel

---

### 8. Retrieval Path Clarity

**Question:** Can the eye still recover something human and meaningful?

| Score | Description |
|-------|-------------|
| 0 | No retrieval path |
| 1 | Almost no anchor |
| 2 | Weak anchor, difficult recovery |
| 3 | Some anchor survives |
| 4 | Strong anchor remains under corruption |
| 5 | The retrieval path is beautifully managed — damaged but compelling |

**What to look for:**
- Eyes
- Mouth
- Silhouette
- Head shape
- Pose
- Hands
- Repeated phrase tied to face/body
- One stable glamour zone

**Red flags:**
- Total subject annihilation
- Texture replaces body
- Unreadable soup

---

### 9. Anti-Drift Integrity

**Question:** Does the image resist common drift into generic territory?

| Score | Description |
|-------|-------------|
| 0 | Fully drifted |
| 1 | Mostly generic with minor glitch cues |
| 2 | Mixed; some originality, some heavy drift |
| 3 | Generally on-track |
| 4 | Clearly resists generic drift |
| 5 | Exceptionally specific; cannot be mistaken for generic cyberpunk/editorial/filter work |

**What to look for:**
- Not just cyberpunk
- Not just beauty ad
- Not just VHS nostalgia
- Not just rainbow effect soup
- Not analog horror cliché unless intentionally hybridized

**Red flags:**
- Fake glitch filter
- Editorial cyberface
- Empty trend aesthetics
- Plugin-core look

---

### 10. Overall Subtype Coherence

**Question:** Does the image clearly belong to a meaningful glitchcore branch or hybrid?

| Score | Description |
|-------|-------------|
| 0 | No subtype identity |
| 1 | Subtype unclear or contradictory |
| 2 | Maybe a subtype, but weakly defined |
| 3 | Subtype mostly coherent |
| 4 | Clear subtype identity |
| 5 | Strong subtype identity with smart hybrid handling if relevant |

**What to look for:**
- Candy broadcast hallucination
- Pastel identity smear
- RGB phantom
- Composite fever dream
- CRT contour
- Ghost-frame body
- Raster meltdown
- Pixel ghost
- Text-screen heartbreak
- Surveillance apparition
- Candy-crash compression
- Glitter-signal overprint

**Red flags:**
- Mishmash with no family resemblance
- Subtype says one thing, image does another
- Unsupported hybrid soup

---

## 4. Score Interpretation

| Score Range | Interpretation |
|-------------|----------------|
| 45–50 | Excellent / Repo-worthy — vivid artifact logic, strong body/image transformation, emotionally alive, specific, memorable, likely a reference example |
| 38–44 | Strong — mostly successful, a few weak spots, good enough to keep and learn from |
| 30–37 | Promising but flawed — useful ideas, likely needs refinement, often missing hierarchy, specificity, or emotional pressure |
| 20–29 | Weak — drift risk high, structure not holding, likely too generic, too muddy, or too safe |
| 0–19 | Failed / not really glitchcore — weak artifact logic, no body stakes, generic style-adjacent image, too abstract or too cosmetic |

---

## 5. Fast Evaluation Version

If you need a quick pass, rate just these five:

1. Figure anchor
2. Lead artifact driver
3. Palette hierarchy
4. Emotional consequence
5. Anti-drift integrity

Score out of 25.

That will catch most bullshit fast.

## 6. Fix Matrix

### Low figure anchor
**Fix with:**
- Stronger face or pose retrieval path
- Clearer silhouette
- Preserve eyes/lips/hands/profile

### Low artifact specificity
**Fix with:**
- Name and strengthen one real failure mode
- Reduce generic noise
- Make the driver visible

### Low artifact hierarchy
**Fix with:**
- Choose one lead driver
- Lower opacity of support effects
- Remove redundant layers

### Low palette architecture
**Fix with:**
- One dominant hue family
- One collision hue
- One accent puncture
- Better value structure

### Low density control
**Fix with:**
- Add quiet zones
- Reduce texture redundancy
- Decide actual density level
- Restore rhythm

### Low emotional consequence
**Fix with:**
- Choose emotional tags explicitly
- Let corruption affect beauty or identity
- Add text, temporal weirdness, or more meaningful damage

### Low media specificity
**Fix with:**
- Clarify source logic: CRT / codec / terminal / CCTV / screenshot / etc.
- Use more medium-specific cues

### Low retrieval path
**Fix with:**
- Sharpen or preserve one major body/face zone
- Reduce clutter in one area

### Low anti-drift integrity
**Fix with:**
- Identify the drift type
- Add uglier or more specific behavior
- Remove trend-only surface cues

### Low subtype coherence
**Fix with:**
- Commit harder to one family
- Or define the hybrid clearly
- Stop mixing incompatible signals randomly

---

## 7. Summary Sentence

The glitchcore evaluation rubric measures whether an image successfully transforms a human figure or identity image through specific artifact logic, controlled density, structured palette behavior, emotional consequence, media/material specificity, and resistance to generic drift.

---

## 8. Short Anchor Rule

Judge it by what the damage does to the body, not by how trendy the effects look.
