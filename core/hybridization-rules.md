# Hybridization Rules

How to combine glitchcore subtypes without creating random effect soup.

## Purpose

This file defines the rules for creating hybrid glitchcore images — combinations of two or more subtypes that produce coherent, meaningful results.

Hybridization is powerful but dangerous. Done right, it creates rich, layered images. Done wrong, it creates visual chaos with no identity.

This file provides the guardrails.

---

## 1. Hybrid Philosophy

### 1.1 The Core Principle

A successful hybrid:
- Has one clear primary subtype
- Uses secondary subtypes as support
- Maintains visual hierarchy
- Creates meaningful tension, not random clutter
- Can be named and described coherently

A failed hybrid:
- Has no clear lead
- Mixes incompatible logics
- Creates equal-volume chaos
- Cannot be described simply
- Feels like "everything at once"

---

### 1.2 The 70/30 Rule

**70% primary subtype**
**30% secondary influence**

This keeps the hybrid coherent while adding richness.

## 2. Hybrid Type Taxonomy

### 2.1 Body + Contamination Hybrids

**Structure:**
- Body-based subtype leads (Ghost-Frame, Pastel Smear)
- Contamination subtype adds texture (Text-Screen, Candy-Crash)

**Examples:**
- Ghost-Frame Body + Text-Screen Heartbreak
  - Motion trails with confessional text debris
  - Kinetic loneliness

- Pastel Smear + Candy-Crash Compression
  - Soft dreamy corruption with internet damage
  - Broken dream aesthetic

**Logic:**
Body provides structure. Contamination provides emotional texture.

---

### 2.2 Palette + Failure Hybrids

**Structure:**
- One subtype provides palette energy
- Another provides failure mode

**Examples:**
- Candy Broadcast palette + Candy-Crash Compression
  - Bright hyperpop colors with file damage
  - Bratty internet-broken beauty

- Pastel Smear palette + Surveillance degradation
  - Soft colors with cold institutional damage
  - Dreamy paranoia

**Logic:**
Palette creates emotional temperature. Failure mode creates texture.

---

### 2.3 Structure + Motion Hybrids

**Structure:**
- Structural subtype leads (CRT Contour, Scanline)
- Motion subtype adds time (Ghost-Frame, Temporal Echo)

**Examples:**
- CRT Contour + Ghost-Frame Body
  - Striped signal anatomy with motion trails
  - Iconic kinetic energy

- RGB Phantom + Temporal Echo
  - Chromatic separation through time
  - Spectral motion

**Logic:**
Structure provides form. Motion provides life.

---

### 2.4 Atmosphere + Text Hybrids

**Structure:**
- Atmospheric subtype creates mood
- Text subtype adds mediated language

**Examples:**
- Pastel Smear + Text-Screen Heartbreak
  - Dreamy romantic atmosphere with chat residue
  - Soft internet intimacy

- Surveillance Apparition + Text-Screen Heartbreak
  - Cold institutional atmosphere with confessional text
  - Monitored confession

**Logic:**
Atmosphere creates emotional space. Text adds narrative debris.

---

### 2.5 Maximal Hybrids

**Structure:**
- Composite Fever Dream as base
- Other subtypes as specific influences

**Examples:**
- Composite + Candy Broadcast emphasis
  - Maximum layers with hyperpop palette
  - Overloaded candy delirium

- Composite + Text-Screen emphasis
  - Maximum layers with confessional text
  - Overloaded internet heartbreak

**Logic:**
Composite provides the pile-up framework. Other subtypes provide specific flavor.

## 4. Hybrid Construction Rules

### 4.1 Step-by-Step Process

**Step 1: Choose Primary Subtype**
- This determines the main visual logic
- This should be 70% of the image character

**Step 2: Choose Secondary Subtype**
- This adds specific texture or flavor
- This should be 30% of the image character
- Must be compatible with primary

**Step 3: Determine Hybrid Type**
- Body + Contamination?
- Palette + Failure?
- Structure + Motion?
- Atmosphere + Text?

**Step 4: Assign Drivers**
- Primary driver from primary subtype
- Secondary drivers mix both subtypes
- Maintain hierarchy (primary drivers stronger)

**Step 5: Resolve Palette**
- Usually follow primary subtype's palette
- May borrow accent from secondary
- Must have clear hierarchy

**Step 6: Define Emotional Tone**
- Hybrid should have coherent emotional logic
- Tension between subtypes can be productive
- Random emotional mixing is not

**Step 7: Name the Hybrid**
- Should be describable in 2-4 words
- Examples: "Dreamy Confession," "Bratty Broadcast," "Spectral Motion"

---

### 4.2 Driver Assignment

```
Hybrid: Pastel Smear + Text-Screen Heartbreak

Primary (Pastel Smear - 70%):
- Primary driver: overprint_stacking
- Secondary: soft_channel_split
- Tertiary: bloom_contamination

Secondary (Text-Screen - 30%):
- Add: text_interface_debris (medium density)
- Add: subtitle_ghosting (light)
- Reduce: compression_haze (light)

Result:
- Soft dreamy overprint base
- Chat debris floating through
- Romantic internet intimacy
```

## 7. Hybrid Drift Detection

### 7.1 Soup Warning Signs

- Cannot name the hybrid simply
- More than 2 subtypes active
- All drivers at equal strength
- No clear visual hierarchy
- Emotional tone contradictory
- "Everything at once" feeling

### 7.2 Fix Strategies

**If soup detected:**
1. Reduce to 2 subtypes maximum
2. Strengthen primary to 70%
3. Weaken secondary to 30%
4. Remove redundant drivers
5. Clarify emotional logic
6. Give it a simple name
